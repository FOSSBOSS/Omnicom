My recognittion of the similarities in communication languages used in PLC has been met with criticism. Its ok, I'll supply docs, and tools anyway.  
Omnicom: A tool to communicate with Every PLC. 

A tool to craft control messages to plcs, of various types. 
the gist is every PLC made in the last 50 years uses the same communication protocol, 
and that while the assumption is that the methods are unique, or proprietary, the methods still work.

What differs is the mapped memory, and the exact commands / instructions. 

goung to be using arrays to keep track of specific brand / command list variations. 
In my brief investigation, the variation is mainly by OEM. Products from the same OEM
have consitent language and instructions.
